Film Junkie.com

vz19513 - Solo Project

1. Running & Access

Run (with debug) using command: DEBUG=website:* npm start

Access via: https://localhost:3000/

It will flag as unsafe due to the self-signed security certificate.

2. Mission Statement

The website is intended to display content for a (fictional) group of film critics. The primary content is film reviews.

With the subject matter, I have attempted to go for a very cinematic look, with a dark background and heavy emphasis on images just like in a cinema.

3. Sections

Index (Homepage)

The main page is intended to showcase the featured review as well as the latest reviews.

About

Tabulated container with photos of and details of each reviewer.

The images are not my own have been sourced from the following links:

  • Male #1
  • Male #2
  • Female #1

Contact

The Contact page is comprised essentially of two elements. A google map widget that shows the location of their office (simply set to UoB at the moment) and a contact form whereby the user can send a message to the reviewers.

When the form is actioned, the data is checked and if it is valid then an INSERT query is actioned and the data is entered into the database.

All Films

This is a page contains a list of films that the group has reviewed along with some key details. This content is pulled from a database containing details of each reviewed film, accessed using SQLite3 and Ajax.

Review Pages

There are web pages for each film review. Each review page will contain a short slideshow including images from the film, rating, reviewer and the written review.

4. Estimated Grades

Self-marking and justification:
  • A for HTML
    • Significant use of HTML in a variety of forms throughout and regular use of W3 validator.
    • Many of the main pages ('About', 'Contact', 'Home'...) have been created using Jade templates to produce HTML.
    • Text manipulation: use of bold, headers, centering, aligning, bulletpoints etc.
    • Creation of containers such as forms or sidebars which are then manipulated by CSS.
  • A for CSS
    • Use of fade in & fade out (on hover) effects for links on the main page.
    • Use of opacity effects (on hover) for images on the main page.
    • Customising text layouts. For example, creation of a two column newspaper-style text layout for the review pages.
  • A for JS
    • Created a tabbed window using a combination of Javascript and CSS on the 'About' page.
    • Created a picture slideshow using Javascript on the film review pages.
  • B for PNG
    • The following are were all acheived using the GIMP image manipulation program:
    • Exporting downloaded image files as .PNG
    • On the 'About' page, these images were cropped so that every one was the same size and then the colours modified to grayscale.
    • On the Home page, images were cropped and exported to fit their respective boxes. In some cases, I used the Levels tool in GIMP to amend the brightness & contrast of the images.
  • C for SVG
    • Use of Inkscape to create the SVG banner header.
    • This is a simple SVG file including text and an altered image. The image was found online, the colour was changed, rotated and then joined with the text.
  • B for Server
    • Set up a server and a router using an express tutorial.
    • Added a self-signed security certificate.
  • B for Database
    • Manually created a SQL databasee: this database contains a table for film details and a table for messages sent via the contact form.
    • SELECT query is utilised to display the film data (on the 'All Films' page.
    • INSERT query is utilised to insert the form data on the contact page into the database.
    • Error messages are produced if fields do not contain data or a success message is produced if input is accepted.
    • Currently, the responses are only in the form of JSON strings
  • B for Dynamic pages
    • Use of Javascript routes to switch between views.
    • Use of Jade templates to create these views. Each of the views from the index inherits it's main characteristics from the 'Layout' template. This makes the website easier to maintain. For example, if the administrator wants to amend a link in the sidebar, they only need to amend the 'Layout' template rather than manually edit every page.
    • Some key elements of the review pages are dynamic. For example, the user needs only set the film title and the relevant images are found and displayed in the slideshow.
