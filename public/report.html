1. Running & Access

Run (with debug) using command: DEBUG=website:* npm start

Access via: https://localhost:3000/

It will flag as unsafe due to the self-signed security certificate - allow insecure content to access.

2. Mission Statement

The website is intended to display content for a (fictional) group of film critics. The primary content is film reviews.

With the subject matter, I have attempted to go for a very cinematic look, with a dark background and heavy emphasis on images just like in a cinema.

3. Sections

Index (Homepage)

The main page is intended to showcase the featured review as well as the latest reviews.

About

A page to explain to the user about who the team are and what their background is. Tabulated container with photos of the team and details of each reviewer.

The images are not my own have been sourced from the following links:

  • Male #1
  • Male #2
  • Female #1

Contact

The Contact page is comprised essentially of two elements. A google map widget that shows the location of their office (simply set to UoB at the moment) and a contact form whereby the user can send a message to the reviewers.

When the form is actioned, the data is checked and if it is valid then an INSERT query is actioned and the data is entered into the database.

Reviews

This is a page contains a list of films that the group has reviewed along with some key details. This content is pulled from a database containing details of each reviewed film, accessed using SQLite3 and Ajax. The images used on these pages are not my own.

Review Pages

There are web pages for each film review. Each review page will contain a short slideshow including images from the film, rating, reviewer and the written review as well as a 'Like' button to like the film.

    • Use of fade in & fade out (on hover) effects for links on the main page.
    • Use of opacity effects (on hover) for images on the main page.

    • Use of CSS to set style and positioning of 'Next' & 'Previous' page buttons.
  • B for JS
    • There is a substantial amount of client-side Javascript throughout my website.

    • Wrote the code for a 'Like' button on the main pages and once clicked, the result is kept in LocalStorage.
    • Included a Google maps widget that displays the company location.

    • The following are were all acheived using the GIMP image manipulation program:
    • Exporting downloaded image files as .PNG

    • The next images on this page (John/James/Sarah) were cropped so that every one was the same size and then the colours modified to grayscale.

    • The following were all created within Inkscape (except the favicon).
    • Creation of the banner header. This is a simple SVG file including text and an altered image. The image was found online, the colour was changed, rotated and then joined with the text.

This makes the website easier to maintain. For example, if the administrator wants to amend a link in the sidebar, they only need to amend the 'Layout' template rather than manually edit every page.
    • The key elements of the review pages are dynamic. For example, the user needs only set the FilmId and the relevant images, links, rating and author will populate.
